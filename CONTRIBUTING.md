# How  to Contribute Next-Aura

Thank you for your interest in contributing to Next-Aura. We welcome contributions from everyone who wants to improve this project.

---

## How to Contribute

1. Fork the repository  
2. Clone your forked repository  
3. Create a new branch  
   - Example: Your Name /  feature/add-login or fix/navbar-bug  
4. Make your changes  
5. Commit your changes with a clear message  
6. Push your branch to GitHub  
7. Create a Pull Request  

---

## Contribution Guidelines

- Write clean and readable code  
- Follow the existing project structure  
- Use meaningful variable and function names  
- Keep commits small and clear  
- Add comments if your code is complex  

---

## Code Style

- Use consistent formatting  
- Follow TypeScript best practices  
- Avoid unnecessary code  
- Reuse components and functions when possible  

---
## Issue and Project Workflow

To keep the project organized, please follow this workflow before starting any work:

1. Go to the Issues section and choose an issue you want to work on  
2. Assign the issue to yourself  
3. Move the issue to the project board  

---

## Project Status Flow

Update the issue status properly while working:

- Ready → When the issue is selected but work has not started yet  
- In Progress → When you are actively working on the issue  
- In Review → When your work is completed and needs review  
- Done → When the issue is fully completed and approved  

---

## Important Notes

- Always assign yourself before starting work  
- Do not work on issues already assigned to someone else  
- Keep the status updated to help team collaboration  

## Pull Request Rules

- Make sure your code works correctly  
- Do not include unnecessary files  
- Write a clear description of your changes  
- Link the related issue if possible  

---

## Communication
Be respectful and constructive when discussing ideas or reviewing code.

---

Thank you for helping improve Next-Aura.
